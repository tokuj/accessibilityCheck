「あなたはプロの〇〇です」をもうやめたい

事実（現状・問題点）

1. 従来の「役割演技プロンプト（ロールプレイ）」の限界
	•	「あなたはプロの〇〇です」のようなプロンプトは、AIに特定の言語・思考スタイルを模倣させるために使われてきた。
	•	しかし、多くのユーザーが同じような「制服」を着せるため、アウトプットがステレオタイプ化し、独自性が失われる。
	•	AIが進化するにつれ、こうした役割指定は効果が希薄化しつつある。
	•	場合によっては、紋切り型のペルソナ模倣が高度な推論を阻害することさえある。 

2. AIが状況を理解することの難しさ
	•	人間同士のコミュニケーションでも、膨大な暗黙知や背景情報が共有された上で成立している。
	•	多くのプロンプトは、こうした暗黙知や文脈を十分に渡さずにタスク遂行だけを求める構造になっており、それがAIとの対話の困難さにつながっている。 

3. 「マニュアル」と「憲法」の違い
	•	旧来のプロンプトは「マニュアル（こうしなさい）」型であり、既知のケースには強いが未知のケースへは弱い。
	•	一方で、意図された基本原則（「憲法」）を与えることで、未知の状況でも判断の指針を示せる可能性がある。
	•	ここでいう「憲法」は偏見ではなく、**望ましい判断を行うための意図的な指針（バイアス）**として定義される。 

⸻

改善案（提案されている方法・実践）

1. コグニティブ・デザイン（認知設計）の導入
	•	役割指定ではなく「世界の捉え方のレンズ」をAIに与える設計を行う。
	•	これは単にタスクを与えるのではなく、AIが長期的・根本的に一貫した思考を展開できるように構造化された枠組みを設計する方法論。 

2. 思考のレンズを設計する5つの要素

以下の要素を明示的にプロンプトに含めることで、AIに文脈・価値観・制約を伝えやすくする：
	1.	前提（Premise）
	•	疑うことのない事実や価値観
	2.	状況（Situation）
	•	現在のコンテキストと変数
	3.	目的（Purpose）
	•	達成したい具体的なゴール
	4.	動機（Motive）
	•	目的を追求する根源的な理由
	5.	制約（Constraint）
	•	守るべきルールや踏み越えない境界線 

3. 実践例に基づくプロンプト改善
	•	旧来プロンプト（例：ただ「プロのコンサルタントです」と指定するだけ）では、背景・読者・優先度等が十分にAIへ伝わらず、一般的・陳腐な回答に終始する可能性がある。
	•	一方で、上記の5要素をプロンプト内に明示することで、AIが一貫したストーリーを生成できるようになる。 

⸻

まとめ
	•	単純なロールプレイ型のプロンプトは、AIの進化に伴い効果が低下している。
	•	AIとの対話設計には、単純な役割指定ではなく、状況・価値観・目的・動機・制約といった認知的要素の設計が重要である。
	•	これらの要素を体系化しプロンプトへ組み込むことで、AIはより一貫性のあるアウトプットを生成可能になる。
