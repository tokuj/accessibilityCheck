# Markdown 修正指示書（markdownlint 全50ルール準拠）

本指示書は markdownlint の全ルール（MD001〜MD050）に基づき、Markdown ファイルを修正する。
.markdownlint.jsonの設定を確認する。.markdownlint.jsonと下記の記述が異なっている場合は.markdownlint.jsonを優先する。
Markdownファイルを受け取ったら下記の方針に従い修正を行う。
文章の改変は一切認めません。文章を改変しないと修正が行えないものは必ずユーザーに確認を行う。

## 1. 見出し（Heading）

このセクションでは見出しに関するルールを示す。

### MD001 見出しレベルは 1 段階ずつ増加させる

`# → ## → ###` の順序を守る。

### MD002 ファイル最初の見出しはレベル1

冒頭に `# タイトル` を配置する。

### MD003 見出しスタイル統一

ATX（`#`）または Setext のどちらかに統一する。

### MD018 `#` の直後のスペース不足を修正

スペースが無い場合は 1 つ追加する。

### MD019 `#` の直後のスペース過多を修正

2 個以上なら 1 個にする。

### MD020 / MD021 閉じ ATX 見出しのスペース修正

閉じ ATX を使う場合のみ内側のスペースを調整する。

### MD022 見出し前後に空行を入れる

前後に空行が無い場合は追加する。

### MD023 見出し行頭にインデントを置かない

行頭の空白やタブを削除する。

### MD024 見出し文言の重複禁止

同じレベルで同一文言を使用しない。

### MD025 レベル1見出しは1回のみ

レベル1見出しは本書タイトルのみとする。
章構造はすべてレベル2（`##`）以下で表現する。

### MD026 見出し末尾の句読点禁止

末尾が `。` や `.` の場合は削除する。

### MD041 最初の項目はレベル1見出しとする

本文が先頭にある場合は `# 見出し` を先頭へ追加する。

## 2. 段落・空行・スペース

このセクションでは空行やスペースの扱いを定義する。

### MD012 空行は連続1行まで

2 行以上続く場合は 1 行に修正する。

### MD008 行末の余分なスペースを削除

### MD009 強制改行用以外の行末スペース禁止

### MD010 タブ禁止

タブをすべてスペースへ変換する。

## 3. リスト（List）

このセクションではリスト構造のルールを示す。
すべてのリストは段落の後に続けて使用され、前後に空行を置く。

### MD004 箇条書き記号統一

`-`、`*`、`+` を混在させない。

### MD005 リストインデント不整合の修正

インデントの深さを揃える。

### MD006 行頭揃えを適正化

字下げし過ぎているリストを修正する。

### MD007 箇条書きインデントを統一

標準は 4 スペースとする。

### MD032 リスト前後に空行を挿入

本書ではすべてのリストを「段落 → 空行 → リスト → 空行 → 段落」の
構造で記述し、MD032 違反を恒久的に回避する。

## 4. コードブロック

このセクションではコードブロックの扱いを定義する。

### MD014 コマンド先頭の `$` を不要なら削除

### MD031 コードブロック前後の空行を追加

### MD040 言語指定の無いコードブロックに言語名を追加

### MD048 コードブロック記法統一

フェンス（```）とチルダ（~~~）を混在させない。

## 5. 引用（Blockquote）

このセクションでは引用構文を扱う。

### MD027 `>` の後はスペース1つまで

### MD028 空白行のみの引用行禁止

引用内部に空白行があれば削除する。

## 6. 強調・リンク・HTML

このセクションではリンク・強調・HTML の扱いを示す。

### MD011 リンク構文誤りの修正

`[text](url)` の形式へ統一する。

### MD033 不要なインライン HTML 禁止

`<b>` や `<font>` を Markdown 構文へ置換する。

### MD034 ベア URL を禁止

`<URL>` または `[text](URL)` に統一する。

### MD036 強調を見出し代わりに使用禁止

`**text**` が見出し的用途の場合は `##` へ変更する。

### MD049 斜体記法統一

`*italic*` または `_italic_` に統一する。

### MD050 太字記法統一

`**bold**` または `__bold__` に統一する。

このセクションでは末尾行の扱いを示す。

### MD047 ファイル末尾に1行の改行を入れる

## 8. 補足メタルール

本指示書内で発生しうるルールの競合を回避する。

### MD002 / MD041 の整合性確保

最初の見出しは必ずレベル1とする。

### MD003 / MD048 の記法整合性

見出し・コードブロック記法を統一する。

### MD004〜MD007 の一括処理

リスト記法は「記号統一」「インデント統一」「前後空行」の
3 点を必ず修正する。

## 9. 修正フロー

このフローに従って Markdown を修正する。

1. 行頭・行末・空白の修正（MD008〜MD010, MD012）
2. 見出し構造の修正（MD001〜MD026, MD041）
3. リスト修正（MD004〜MD007, MD032）
4. コードブロック修正（MD014, MD031, MD040, MD048）
5. 引用修正（MD027, MD028）
6. 強調・リンク・URL修正（MD011, MD033, MD034, MD036, MD049, MD050）
7. ファイル末尾の改行（MD047）
8. 全体読み合わせ
